## 문제 1 — localStorage 헬퍼 함수 중복 (🔴 심각)

**원인:** 각 페이지 파일 상단에 localStorage read/write 함수를 직접 정의하여 동일한 함수가 5개 파일에 복붙되어 있음.

| 중복 함수명 | 발견된 파일 | 비고 |
|------------|-----------|------|
| `loadOwnedHeroes()` / `saveOwnedHeroes()` | `GamePage.tsx`, `OffenseGamePage.tsx`, `HeroesPage.tsx` | 3중 중복 |
| `loadRaidStageProgress()` / `saveRaidStageProgress()` | `GamePage.tsx`, `AIGamePage.tsx` | 2중 중복 |
| `loadWallTalents()` / `saveWallTalents()` | `GamePage.tsx`, `AIGamePage.tsx` | 2중 중복 |
| `loadStageProgress()` / `saveStageProgress()` | `OffenseGamePage.tsx`, `HeroesPage.tsx` | 2중 중복 |
| `loadProtagonistSave()` | `HeroesPage.tsx` → 다른 파일들이 import | 출처 파일 부적절 |
| `loadManghonguSave()` / `saveManghonguData()` | `game/manghonguData.ts` | 분리 불일치 |

**문제점:**
- 함수 하나를 수정하면 모든 복붙 위치에서 따로 수정해야 함
- 키 이름 오타 발생 시 특정 파일에서만 버그 발생
- `HeroesPage.tsx`가 유틸리티 역할을 겸하는 기형 구조

**권장 해결:** `frontend/src/utils/localStorage.ts` 단일 파일로 통합

## 문제 7 — 백엔드 `req: any` 패턴 전체 (🟠 중간)

NestJS에서 JWT로 인증된 요청의 `req.user`를 접근할 때 거의 모든 컨트롤러에서 `req: any`를 사용하고 있음.

**영향 파일:**
- `auth.controller.ts`
- `achievement.controller.ts`
- `leaderboard.controller.ts`
- `tournament.controller.ts`
- `user.controller.ts`
- `friend.controller.ts` (추정)
- `shop.controller.ts` (추정)
- `game.gateway.ts`

**패턴 통일 방법:** 프로젝트 공통 타입 정의 파일 생성

```typescript
// backend/src/types/auth.types.ts (신규 생성)

  user: { id: number; email: string; };
}
```
