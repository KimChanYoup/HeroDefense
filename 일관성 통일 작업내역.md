## 작업 1 — 백엔드 공통 인증 타입 생성

### 신규 파일 생성
**`backend/src/types/auth.types.ts`** (새 파일)

```typescript
export interface AuthUser {
  id: number;
  email: string;
  username: string;
}

export interface AuthenticatedRequest {
  user: AuthUser;
}
```

### 문제
- 거의 모든 컨트롤러에서 `@Request() req: any`를 사용
- `req.user.id`, `req.user.username` 접근 시 타입 안전성 없음
- JWT 인증된 요청에 대한 타입 정의가 전혀 없었음

### 수정 내용
아래 7개 컨트롤러 전체에서 `req: any` → `req: AuthenticatedRequest` 교체:

| 파일 | 교체 건수 |
|------|----------:|
| `backend/src/auth/auth.controller.ts` | 1 |
| `backend/src/user/user.controller.ts` | 13 |
| `backend/src/achievement/achievement.controller.ts` | 2 |
| `backend/src/leaderboard/leaderboard.controller.ts` | 1 |
| `backend/src/tournament/tournament.controller.ts` | 5 |
| `backend/src/friend/friend.controller.ts` | 8 |
| `backend/src/shop/shop.controller.ts` | 2 |
| **합계** | **32** |

추가로 `user.controller.ts`의 `saveData: any` 2건도 `saveData: unknown`으로 교체.

## 작업 2 — 프론트엔드 localStorage 헬퍼 함수 통합

### 문제
동일한 localStorage 읽기/쓰기 함수가 5개 파일에 중복 정의되어 있었음:
- `GamePage.tsx`: `loadOwnedHeroes`, `saveOwnedHeroes`, `loadWallTalents`, `loadRaidStageProgress`, `saveRaidStageProgress`, `isRaidStageUnlocked`
- `AIGamePage.tsx`: `loadWallTalents`, `loadRaidStageProgress`, `saveRaidStageProgress`, `isRaidStageUnlocked`, `loadAIFactionStars`, `saveAIFactionStars`
- `OffenseGamePage.tsx`: `loadOwnedHeroes`, `loadStageProgress`, `saveStageProgress`, `isStageUnlocked`
- `HeroesPage.tsx`: `loadSave`, `writeSave`, `loadProtagonistSave`, `writeProtagonistSave`, 등 13개 함수 (export 형태)

### 신규 파일 생성
**`frontend/src/utils/localStorage.ts`** (새 파일, 약 240줄)

포함된 함수 목록:

| 함수 | 설명 |
|------|------|
| `loadSave` / `writeSave` | 영웅 전체 세이브 데이터 |
| `loadProtagonistSave` / `writeProtagonistSave` | 주인공 일반 세이브 |
| `loadProtagonistDefenseSave` / `writeProtagonistDefenseSave` | 주인공 디펜스 세이브 |
| `loadProtagonistAISave` / `writeProtagonistAISave` | 주인공 AI 모드 세이브 |
| `loadProtagonistOffenseSave` / `writeProtagonistOffenseSave` | 주인공 오펜스 세이브 |
| `loadProtagonistRaidSave` / `writeProtagonistRaidSave` | 주인공 레이드 세이브 |
| `loadOwnedHeroes` / `saveOwnedHeroes` | 보유 영웅 목록 |
| `loadRaidStageProgress` / `saveRaidStageProgress` / `isRaidStageUnlocked` | 레이드 스테이지 진척도 |
| `loadStageProgress` / `saveStageProgress` / `isStageUnlocked` | 오펜스 스테이지 진척도 |
| `loadWallTalents` / `saveWallTalents` | 벽 탤런트 |
| `loadAIFactionStars` / `saveAIFactionStars` | AI 팩션 별 점수 |
| `loadManghonguSave` / `saveManghonguData` | 망혼구 세이브 |

타입 정의도 함께 이동:
- `HeroSaveData`, `AllHeroSave`
- `ProtagonistSaveData`, `ProtagonistDefenseSaveData`, `ProtagonistAISaveData`
- `RaidStageProgress`, `StageProgress`

### 수정된 파일

**`frontend/src/pages/GamePage.tsx`**
- 중복 함수 6개 제거 (`loadOwnedHeroes`, `saveOwnedHeroes`, `loadWallTalents`, `loadRaidStageProgress`, `saveRaidStageProgress`, `isRaidStageUnlocked`)
- `HeroesPage` import → `utils/localStorage` import로 변경
- 불필요한 주석 ("Triggering Vite refresh...") 제거

**`frontend/src/pages/AIGamePage.tsx`**
- 중복 함수 8개 제거 (`loadWallTalents`, `loadRaidStageProgress`, `saveRaidStageProgress`, `isRaidStageUnlocked`, `loadAIFactionStars`, `saveAIFactionStars` 등)
- `HeroesPage` import → `utils/localStorage` import로 변경

**`frontend/src/pages/OffenseGamePage.tsx`**
- 중복 함수 4개 제거 (`loadOwnedHeroes`, `loadStageProgress`, `saveStageProgress`, `isStageUnlocked`)
- 중복 주석 ("── 스테이지 시스템 ───") 1개 제거
- `HeroesPage` import → `utils/localStorage` import로 변경

**`frontend/src/pages/HeroesPage.tsx`**
- 파일 상단 150줄 분량의 함수/타입 정의 제거
- `utils/localStorage`에서 import 후 re-export하는 구조로 변경
- 기존에 이 파일에서 직접 import하던 다른 파일들(`GamePage`, `AIGamePage`, `OffenseGamePage`)과의 호환성을 위해 re-export 유지

## 미완료 항목 (추가 작업 필요)

아래 항목들은 범위가 크거나 기능에 직접 영향을 줄 수 있어 별도 작업으로 분리:

| 항목 | 이유 |
|------|------|
| 프론트엔드 `any` 타입 157건 제거 | 각 파일별 타입 설계 필요, 실수 시 런타임 오류 가능 |

| 에러 처리 패턴 통일 | 서비스 로직 이해 필요 |
| 대형 파일 분할 (GameEngine.ts 등) | 고위험 리팩토링, 별도 계획 필요 |
