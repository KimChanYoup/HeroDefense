## 🌐 IV.1 Web 모듈

### Major (2점짜리)

#### ✅ [구현완료] 프레임워크: 프론트엔드 + 백엔드 동시 사용 — **2점**
- **조건**: 프론트엔드 프레임워크(React/Vue/Angular/Svelte 등) **하나** + 백엔드 프레임워크(Express/NestJS/Django 등) **하나**를 모두 사용
- **단, Full-stack 프레임워크(Next.js, Nuxt.js, SvelteKit) 하나로 프론트+백엔드 모두 처리하면 2점 동시 인정**
- **우리 구현**: React (프론트) + NestJS (백엔드)
- **파일**: `frontend/src/` (React) + `backend/src/` (NestJS)
- ⚠️ **주의**: 프론트만 또는 백엔드만 쓰면 각 1점(Minor)짜리로 분리됨. 둘 다 쓰면 2점 Major 하나로 처리.

---

#### ✅ [구현완료] 실시간 기능 (WebSocket) — **2점**
- **조건**:
  - 클라이언트 간 실시간 업데이트
  - 연결/해제 graceful 처리
  - 효율적인 메시지 브로드캐스팅
- **우리 구현**: Socket.io `/ws` 네임스페이스로 로비 생성, 게임 상태 브로드캐스트, 플레이어 액션 릴레이, 인게임 채팅, 재연결 로직
- **파일**: `backend/src/game/game.gateway.ts`, `frontend/src/pages/LobbyPage.tsx`, `GamePage.tsx`

---

#### ✅ [구현완료] 유저 인터랙션 (채팅 + 프로필 + 친구) — **2점**
- **조건**:
  - 기본 채팅 시스템 (사용자 간 메시지 송수신)
  - 프로필 시스템 (유저 정보 조회)
  - 친구 시스템 (친구 추가/삭제, 친구 목록)
- **우리 구현**:
  - 친구 간 1:1 DM 채팅 (`GET/POST /friends/:id/messages`)
  - 게임 로비 실시간 채팅
  - 프로필 페이지 (조회/수정/아바타 업로드)
  - 친구 추가/수락/거절/삭제, 온라인 상태 표시
- **파일**: `frontend/src/pages/FriendsPage.tsx`, `ProfilePage.tsx`, `backend/src/friend/`

---

#### ❌ [미구현] 공개 API (Public API with API Key) — **2점**
- **조건**:
  - API 키로 보안된 퍼블릭 API
  - Rate limiting (요청 수 제한)
  - 문서화 (Swagger 등)
  - GET / POST / PUT / DELETE 최소 5개 엔드포인트
- **우리 구현**: ThrottlerModule로 전체 rate limiting은 있지만 API 키 기반 퍼블릭 API 없음
- **획득 가능**: 구현 시 +2점
