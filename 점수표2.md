## 1. 점수 계산 검증 — 19점인가, 24점인가?

### 결론: 총 클레임은 23점, 평가 최대치는 19점 (24점 아님)

**점수 체계 (PDF 기준):**
- 14점 이상 검증 → 통과 (기본 100점)
- 14점 초과분 → 보너스 (최대 5점)
- **절대 최대치: 14 + 5 = 19점**

**우리 클레임 계산:**

| 구분 | 항목 | 분류 | 점수 |
|------|------|------|-----:|
| Base | 프레임워크 (React + NestJS) | Web Major | 2 |
| Base | 실시간 WebSocket | Web Major | 2 |
| Base | 유저 인터랙션 (채팅+프로필+친구) | Web Major | 2 |
| Base | 표준 사용자 관리 | UM Major | 2 |
| Base | 웹 기반 게임 | Gaming Major | 2 |
| Base | 원격 플레이어 | Gaming Major | 2 |
| Base | ORM (Prisma) | Web Minor | 1 |
| Base | 국제화 (한/영/일) | Access Minor | 1 |
| Base | 게임 커스터마이징 | Gaming Minor | 1 |
| Base | 게임화 시스템 | Gaming Minor | 1 |
| **Base 합계** | | | **16** |
| Bonus | AI 대전 상대 | AI Major | 2 |
| Bonus | 멀티플레이어 3+ | Gaming Major | 2 |
| Bonus | 토너먼트 시스템 | Gaming Minor | 1 |
| Bonus | GDPR 컴플라이언스 | Data Minor | 1 |
| Bonus | 헬스체크 + 자동 백업 | DevOps Minor | 1 |
| **Bonus 합계** | | | **7** |
| **총 클레임** | | | **23** |

**보너스 캡 계산:**
```
총 클레임 23점 - 필수 기준 14점 = 9점 초과
보너스 = min(9, 5) = 5점 (캡 적용)
최종 평가 점수 = 14 + 5 = 19점
```

**→ 24점은 없음. 총 클레임이 23점이고, 시스템 최대치가 19점.**

## 5. 도미노 시나리오 — 최악의 경우

웹 기반 게임 모듈이 "협동 게임이라 PvP 조건 미달"로 reject되면:

```
웹 기반 게임 reject (2점)
    ↓ 의존성 붕괴
원격 플레이어 reject (2점)
멀티플레이어 3+ reject (2점, 보너스)
AI 대전 상대 reject (2점, 보너스)
토너먼트 reject (1점, 보너스)
    ↓
Base 잔여: 16 - 2 - 2 = 12점 → 14점 미달 → 탈락
```

**→ 가장 중요한 방어 포인트는 "웹 기반 게임" 모듈 정당화**
